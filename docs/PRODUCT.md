# Imprint v2 — Продуктовый документ

> *Твоя жизнь на карте.*

**Статус:** зафиксировано 05.07.2026. Источник правды для разработки. Изменения — только через обновление этого файла.

---

## 1. Позиционирование

**Imprint — личный дневник жизни на карте с элементами соцсети.**

Смена курса относительно v1: раньше — геосоцсеть (Live Map, Discovery, Echoes), теперь — **diary-first**. Продукт обязан быть полноценным и приносить ценность даже пользователю с нулём друзей в приложении. Социальный слой — надстройка, а не фундамент.

Иерархия ценности:
1. **Ядро:** моя карта, мои воспоминания (пины), мои главы, мой путь во времени.
2. **Соцслой v1:** шаринг ссылкой, подписки, реакции.
3. **Не в v1:** Live Map, live presence, Discovery незнакомцев, Echoes, лента.

Платформа v1: **веб (desktop-first, адаптив под мобильный браузер)**. Мобильное приложение — после стабилизации веба.

## 2. Ключевые сущности

| Сущность | Описание |
|---|---|
| **Pin (воспоминание)** | Событие, привязанное к точке на карте: заголовок, текст, фото/видео, дата события (`pinned_at`), видимость. По умолчанию **private**. |
| **Chapter (глава)** | Коллекция пинов, объединённая периодом/темой («Год в Лиссабоне»). Имеет цвет, обложку, описание, даты. Профиль строится из глав. |
| **Trail (путь)** | Маршрут главы: автоматически строится по хронологии пинов, редактируется вручную (промежуточные точки, стиль сегментов: сплошная/пунктир, аннотации на карте). |
| **Profile** | Публичная витрина: главы, которыми пользователь решил поделиться. |

## 3. Функциональный скоуп v1 (сайт)

### 3.1 Ядро (дневник)
- **Memory Map** — личная карта со всеми пинами; кластеризация при зум-ауте; 3D-глобус на дальнем зуме.
- **Timeline ⟷ Map** — главная фича и wow-момент. Лента событий синхронизирована с картой: скролл ленты → камера летит к месту; слайдер времени → «проигрывание» пути по карте.
- **CRUD пинов** — создание кликом по карте или из формы (поиск места, геокодинг), фото/видео, дата события ≠ дата создания.
- **Chapters** — CRUD, назначение цвета, привязка пинов, два вида: карта-вид (путь+пины) и story-вид (прокручиваемая «книжная» история с вставками мини-карт).
- **Trails** — автопостроение пути главы + ручное редактирование, стили сегментов, аннотации.
- **Импорт прошлого** — загрузка фото → EXIF (гео+дата) → черновики пинов; импорт GPX. (Google Timeline — v1.5.)
- **On this day** — блок «в этот день год назад».

### 3.2 Соцслой
- **Шаринг ссылкой:** публичная страница главы и пина (для visibility=public), красивые OG-превью.
- **Подписки:** follow/unfollow, списки подписчиков/подписок.
- **Реакции** на публичные и friends-пины.
- Видимость: `private` (default) / `friends` / `public`.

### 3.3 Инфраструктура v1
- Auth: email + Google OAuth (Supabase Auth).
- Онбординг: регистрация → первая глава → первый пин (или импорт фото) → готовая мини-карта за 5 минут.
- Настройки: профиль, приватность, экспорт своих данных, удаление аккаунта.

### 3.4 Backlog (v1.5 / v2)
Google Timeline import, комментарии, Echoes, годовые итоги («маршрут года» как шаримая анимация), совместные главы, Discovery, мобильное приложение, Time Machine.

## 4. Принципы приватности (нерушимые)

1. Всё создаётся **private по умолчанию**.
2. `friends`-видимость **не раскрывает точные координаты** — только размытую зону (~1 км) либо название места.
3. RLS на каждой таблице. Ни один публичный запрос не должен мочь достать приватную геометрию.
4. Экспорт и полное удаление данных — обязательные функции v1.
5. Никакого перманентного трекинга перемещений: только события, созданные пользователем осознанно.

## 5. Дизайн-принципы

- **Карта — главный герой.** Интерфейс — тонкие слои поверх карты, не наоборот.
- **Тёмная тема по умолчанию**, фирменный кастомный стиль карты (Mapbox Studio). Светлая — later.
- **Меньше ленты — больше пространства.** Никакого бесконечного скролла как основы UX.
- **Движение как язык:** плавные полёты камеры, анимированное «прорисовывание» путей, пульс пинов.
- Референсы: Apple Maps (карточки/анимации), Polarsteps (повествование путешествий), Strava (тёмная карта, маршруты), Airbnb (bottom sheets), Linear (общий уровень полировки UI).

Детальная дизайн-система — отдельный документ `DESIGN.md` (следующий этап).

## 6. Технический стек

| Слой | Технология | Комментарий |
|---|---|---|
| Web | Next.js 15+ (App Router), React 19 | SSR для публичных страниц (шаринг, SEO) |
| Язык | TypeScript strict | `any` запрещён |
| Стили | Tailwind CSS v4 | + свои дизайн-токены |
| Карта | **Mapbox GL JS v3** | кастомный стиль, глобус, анимации; mapbox-gl-draw для редактирования путей. План Б — MapLibre |
| Backend | Supabase: PostgreSQL + **PostGIS**, Auth, Storage, RLS | **v1 — локально (`supabase start`)**, облачный проект — перед деплоем |
| State | Zustand (UI) + TanStack Query v5 (server state) | |
| Медиа | Supabase Storage в v1 | Cloudinary — если упрёмся в трансформации |
| Монорепо | Turborepo + pnpm | web-only на старте, структура готова к добавлению mobile |

### Структура нового репозитория
```
imprint/
├── apps/web/            # Next.js
├── packages/
│   ├── types/           # общие TS-типы
│   ├── api/             # Supabase-клиент, запросы, мапперы
│   └── ui/              # дизайн-система (пока web-компоненты)
├── supabase/            # миграции, RLS, seed, config.toml
├── docs/                # PRODUCT.md, DESIGN.md, ARCHITECTURE.md
└── .cursor/rules/       # правила для Cursor (англ.)
```

### Что наследуем из старого репо (как отправную точку, с ревизией)
- Схему БД: `users`, `chapters`, `memory_pins`, `follows`, `reactions` (+ RLS-подход). Убираем: `live_presence`, `echo_logs`.
- Продуктовые правила приватности.
- Подход к `.cursor/rules` (перепишем под новый скоуп).
