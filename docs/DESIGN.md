# Imprint — DESIGN.md (Dark Atlas)

> Направление утверждено 05.07.2026: **«Тёмный атлас»** — глубокий ночной UI, премиальность, карта как звёздное небо городов, янтарный путь жизни.

Этот документ — источник правды для UI. Все значения переносятся в Tailwind-токены (`packages/ui`) один в один.

---

## 1. Дизайн-принципы

1. **Карта — главный герой.** UI — тонкие слои поверх карты: панели, карточки, шиты. Карта никогда не бывает «превью в углу» на основных экранах.
2. **Тьма как холст.** Тёмная тема — единственная в v1. Свет на экране = контент пользователя (пины, путь, фото).
3. **Один акцент.** Янтарь — единственный системный акцент (действия, активный путь, прогресс). Все остальные цвета принадлежат главам пользователя.
4. **Бордеры вместо теней.** Глубина задаётся уровнями фона и полупрозрачными бордерами, тени почти не используются.
5. **Движение = перемещение.** Каждая анимация метафорически про путь: камера летит, путь прорисовывается, время проматывается.
6. **Книжность в повествовании.** Story-вид главы — как разворот книги: сериф, крупные заголовки, воздух, поля.

## 3. Типографика

| Роль | Шрифт | Начертания | Где |
|---|---|---|---|
| UI / текст | **Inter** (variable) | 400, 500 | Весь интерфейс, тексты историй |
| Display | **Fraunces** (variable, opsz) | 400, 500 | Названия глав, hero-заголовки, лендинг |
| Mono | **JetBrains Mono** | 400 | Даты на слайдере, координаты, статистика |

Шкала: `11 / 12 / 14 (base) / 16 / 20 / 26 / 34 / 44`. Межстрочный: UI — 1.4; тексты историй — 1.75. Заголовки глав: Fraunces, letter-spacing normal; оверлайны (даты-периоды): 11px, letter-spacing 0.12em, цвет главы или янтарь.

Правила: максимум два начертания (400/500), никакого 600+; sentence case во всех кнопках и заголовках UI.

---

## 4. Геометрия и поверхность

- Радиусы: контролы `8px`, карточки `10px`, шиты/модалки `12px`, пилюли `999px`.
- Отступы: шкала 4 → `4/8/12/16/24/32/48`.
- Бордеры `0.5–1px`. Тени — только у плавающих поверх карты элементов: `0 8px 24px rgb(0 0 0 / 0.35)`.
- Оверлеи поверх карты: фон `night-900` на 90% + `backdrop-blur(12px)` — единственное место, где разрешён блюр.
- Контейнер контента story-вида: `max-width 640px`, по центру.

## 5. Карта (кастомный стиль Mapbox)

Создаётся в Mapbox Studio на базе Dark v11, имя стиля `imprint-night`:

| Слой | Цвет |
|---|---|
| Фон/земля | `#0C1226` |
| Вода | `#0E1B33` |
| Крупные дороги | `#1B2545` |
| Мелкие дороги | `#182140` |
| Границы админ. | `#161F3C` |
| Парки/зелень | `#101C2E` |
| Подписи городов | `#5A648A`, крупные — `#8B93B0` |
| POI | скрыты (кроме zoom ≥ 15) |

Здания 3D — выключены. На дальнем зуме — глобус (`projection: globe`), атмосфера в тон `night-950`, звёзды выключены.

**Пины:** круг 12px цвета главы + внешнее кольцо 2px `night-850`; активный — 16px + мягкий пульс (масштаб 1→1.25, opacity-кольцо). Кластеры: круг `night-800` с цифрой `ink-primary` и кольцом цвета доминирующей главы.

**Путь (Trail):** линия 2.5px цвета главы; сегменты-перелёты — пунктир `6 4`; прорисовка анимацией line-dasharray при открытии главы (~1200 мс). Направление — маленькие шевроны по линии на zoom ≥ 10.

## 6. Движение

| Паттерн | Параметры |
|---|---|
| Полёт камеры к пину | `flyTo`, 600–900 мс, curve 1.4 |
| Прорисовка пути | 1200 мс, ease-out, после посадки камеры |
| Появление карточек/шитов | translateY 12px + fade, 250 мс, ease-out |
| Hover карточек | border → `line-strong` + фон `night-700`, 150 мс |
| Пульс активного пина | 1.8 с цикл, бесконечно, только у активного |
| Слайдер времени (play) | пины появляются fade+scale по хронологии, камера следует за «сейчас» |
| Переходы страниц | crossfade 200 мс; карта персистентна между роутами (не перемонтируется) |

Уважать `prefers-reduced-motion`: все полёты → мгновенный `jumpTo`, пульсы выключены.

## 7. Ключевые экраны v1 (инвентарь)

1. **Map + Timeline** (главный): карта на весь экран; слева панель 300px — переключатель «Все воспоминания / глава», таймлайн с точками цвета глав; снизу по центру — слайдер времени с play. Утверждён мокапом (concept A).
2. **Story-вид главы**: hero-обложка (карта-путь + серифный заголовок) → текст 640px → фото-сетки → инлайн мини-карты пинов (раскрываются). Утверждён мокапом.
3. **Создание/редактирование пина**: клик по карте → шит справа: место (геокодер), дата события, заголовок, текст, медиа, глава, видимость (private по умолчанию).
4. **Редактор Trail**: режим поверх карты — перетаскивание промежуточных точек, стиль сегмента, текстовые аннотации на карте.
5. **Профиль**: обложка-глобус со всеми публичными путями + сетка обложек глав («полка книг»), счётчики подписок.
6. **Онбординг**: 3 шага — регистрация → первая глава (имя+цвет) → первый пин или импорт фото (EXIF). Цель: «живая» карта за 5 минут.
7. **Импорт**: дропзона фото → таблица распознанных гео/дат → массовое создание пинов-черновиков.
8. **Настройки**: профиль, приватность, экспорт данных, удаление аккаунта.
9. **Публичные страницы** (SSR): глава `/u/[username]/c/[chapter]`, пин `/p/[id]` — story-вид + OG-превью с картой пути.

Состояния для каждого экрана: empty (приглашение, не извинение), loading (скелетоны `night-700`), error.
