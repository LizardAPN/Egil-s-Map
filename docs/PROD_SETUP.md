# Production: ориентиры

Конкретные хостинг-провайдеры не зафиксированы в репозитории; общий чеклист:

## Supabase (cloud)

- Отдельный production-проект (или environment) с собственными ключами.
- Проверить **RLS** на всех таблицах в публичной схеме.
- Включить бэкапы и политику хранения логов по требованиям продукта.
- Ограничить CORS и разрешённые рефереры для web-клиента.
- Никогда не публиковать `service_role` в клиентских билдах.

## Web (Next.js)

- Переменные окружения в панели хостинга (Vercel / self-hosted и т.д.).
- `NEXT_PUBLIC_*` попадают в браузер — только публичные значения.
- Включить HTTPS, настроить домен `NEXT_PUBLIC_APP_URL`.

## Mobile

- Сборки через EAS или внутренний CI; секреты — в защищённом хранилище.
- В стор-листингах указать политику приватности и использование геолокации/карт.
- Для Google/Apple auth настроить OAuth providers в Supabase и redirect `imprint://auth/callback`.

## Карты и соблюдение приватности

- Live-режим: не хранить полные треки перманентно; соблюдать продуктовые правила размытия для `friends`.
- При изменении правил приватности — синхронно обновлять клиент и RLS.
